feat(infra): 8 Postgres (media 5448), backup all-8, K8s base (auth/messaging/media), Caddy LB + xcaddy HTTP/3, strict TLS/mTLS

Why
---
- Platform has eight domain DBs (auth, listings, bookings, messaging, notification, trust, analytics, media); backup and restore must include media (5448).
- App services (auth, messaging, media) must be in K8s base so Colima/k3s apply deploys them; media needs HTTP 4018 and gRPC 50068.
- Caddy must use LoadBalancer (no hostPort) so two replicas can run on one node (no anti-affinity); Caddy must be built with xcaddy for proper HTTP/3 (QUIC).
- Preflight and k6 require Caddy (2 pods) and Envoy (1 pod) with strict TLS/mTLS and certs loaded; namespaces ingress-nginx, envoy-test, off-campus-housing.

How to test
-----------
- Backup all 8: PGPASSWORD=postgres ./scripts/backup-all-dbs.sh  # -> backups/all-8-YYYYMMDD-HHMMSS/
- Restore: RESTORE_BACKUP_DIR=latest ./scripts/bring-up-external-infra.sh  or  ./scripts/restore-external-postgres-from-backup.sh backups/all-8-<timestamp>
- Full stack: RESTORE_BACKUP_DIR=latest ./scripts/bring-up-cluster-and-infra.sh
- Caddy + Envoy strict TLS: ./scripts/ensure-caddy-envoy-strict-tls.sh  then  CADDY_USE_LOADBALANCER=1 ./scripts/rollout-caddy.sh
- Build Caddy (xcaddy HTTP/3): docker build -t caddy-with-tcpdump:dev -f docker/caddy-with-tcpdump/Dockerfile .
- Build and load media/messaging: docker build -f services/media-service/Dockerfile -t media-service:dev . ; docker save media-service:dev | colima ssh -- docker load ; kubectl apply -k infra/k8s/base
- Preflight: ./scripts/run-preflight-scale-and-all-suites.sh
