feat(ci): production-grade CI + pnpm fix, auth restore, Kafka topics; how to bring up DBs, Redis, Kafka (TLS/mTLS)

Why
---
- CI previously had no Postgres, no Kafka/Redis/MinIO, and no deterministic DB state; pnpm was missing on PATH.
- Integration tests were placeholders; no validation of outbox, rate limit, spam, or media upload.
- Moving to production-style CI ensures every push is validated against real infra and real schemas.

CI workflow (.github/workflows/ci.yml)
--------------------------------------
- pnpm: use pnpm/action-setup@v4 (version from packageManager) before setup-node so pnpm is on PATH; setup-node cache: "pnpm".
- Services: Postgres 16 (5441), Redis 7 (6380), Bitnami Kafka KRaft (9092), MinIO (9000).
- One Postgres in CI: all 8 DBs (auth, listings, bookings, messaging, notification, trust, analytics, media) on port 5441.
- Steps: checkout → pnpm/action-setup → setup-node → install deps → postgresql-client → wait Postgres → bootstrap all 8 (infra/db) → verify-bootstrap → restore auth dump if present (backups/5437-auth.dump or .gz/.zip) → wait Kafka and create event topics (dev.*.events, dev.messaging.dlq) → unit tests → integration tests (A–D) → event-layer-verification → k6 install + optional smoke (|| true).
- Env: REDIS_URL, all *_DB_PORT=5441, VERIFY_DB_PORT=5441.

Bootstrap and verify (single-Postgres support)
-----------------------------------------------
- scripts/bootstrap-all-dbs.sh: optional AUTH_DB_PORT … MEDIA_DB_PORT (defaults 5441–5448); BOOTSTRAP_SKIP_MEDIA=1 to skip media (7-DB-only). CI sets all to 5441. Includes bootstrap_media (8th DB).
- scripts/verify-bootstrap.sh: VERIFY_DB_PORT or per-DB VERIFY_*_PORT; optional media DB check.

Stronger integration tests (A–D)
---------------------------------
- A) Messaging outbox: insert conversation + message + outbox_events; assert COUNT(*) >= 1 (messaging-flow.integration.test.ts).
- B) Media upload: createUploadUrl → mock S3 objectExists → completeUpload → assert status === 'uploaded' (media-upload.integration.test.ts).
- C) Rate limit: checkAndIncrement 30× then 31st throws RATE_LIMIT_EXCEEDED; real Redis (REDIS_URL).
- D) Spam: insert high trust.user_spam_score; assert row readable (403 when Trust check wired).

How to test the setup (bring up DBs, Redis, Kafka)
--------------------------------------------------
- Bring up Redis + Kafka (strict TLS/mTLS) and Postgres (7 instances 5441–5447):
  ./scripts/bring-up-external-infra.sh
  (Requires certs/kafka-ssl for Kafka: PLAINTEXT 9092, SSL 9093 → host 29094; Redis 6380.)
  Optional: SKIP_KAFKA=1 if certs not ready; RESTORE_BACKUP_DIR=latest to restore all 7 from backups/all-7-*.

- After bring-up: bootstrap all 8 DBs from infra/db (or 7 if no media Postgres):
  PGPASSWORD=postgres ./scripts/bootstrap-all-dbs.sh
  Single DB: BOOTSTRAP_ONLY=messaging PGPASSWORD=postgres ./scripts/bootstrap-all-dbs.sh

- Auth dump (5437 → 5441): put 5437-auth.dump (or .gz/.zip) in backups/, then:
  docker compose up -d postgres-auth   # or already up via bring-up-external-infra
  PGPASSWORD=postgres ./scripts/restore-auth-db.sh

- Optional one-shot after bring-up (bootstrap all 8 or 7 if no 5448, then restore auth if dump exists):
  PGPASSWORD=postgres ./scripts/bootstrap-after-bring-up.sh

- Deploy + smoke (k3s):
  ./scripts/deploy-dev.sh
  ./scripts/smoke-test-dev.sh

Optional perf-smoke job (disabled)
----------------------------------
- perf-smoke job if: false; enable when app runs in CI for k6-messaging-limit-finder (fail if P99 > 500ms or error rate > 5%).
