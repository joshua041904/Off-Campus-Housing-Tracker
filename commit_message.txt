feat(och): edge Grafana/contracts/Kafka metrics + CI Postgres/Redis, HTTP/2 matrix threshold, QUIC gate

WHY
---
  • Port-forward is a debug tool; Grafana should ride the same TLS/HTTP2/HTTP3 edge as the app.
  • Unknown /api routes should be 404, not 401, when no service owns the prefix.
  • Invalid listing IDs must be 400 before Postgres ::uuid throws 500.
  • HTTP listing create must emit the same ListingCreatedV1 Kafka envelope as gRPC so analytics can
    project daily_metrics (E2E system-integrity expects new_listings to move).
  • Docker builds must not pick up local *.tsbuildinfo (incremental emit gaps).
  • Prometheus SLO rules must use prom-client label `code`, not fictional `status`.
  • Dev machines without standalone `kustomize` should still apply overlays via kubectl.
  • GitHub Actions runners are not your Mac: integration tests default to 127.0.0.1:5441–5448 and
    Redis 6380 — without service containers those connections are ECONNREFUSED.
  • The protocol-anomaly job rebuilds from committed protocol-comparison.csv; a fixed 3× H2/H1 ratio
    fails on noisy matrix rows (e.g. media/messaging) without re-running the full transport study.
    Parameterizing the threshold keeps CI methodical; default local behavior stays 3×.
  • QUIC hostname invariant should run even when the test matrix is red; it must not sit behind
    needs: transport-validation (skipped when tests fail).
