WHY (cluster-level resource contention — not “fix listings replicas”)
----------------------------------------------------------------------
Messaging / analytics / listings can look “sick” in a **long suite** while the **same paths are fine in isolation**:
  • Back-to-back k6 (especially **constant-arrival-rate**), high VUs, shared Colima/k3s CPU, virtiofs + hypervisor scheduling.
  • **Envoy** retains connection state; **Postgres** spikes when many services load together.
  • **Listings “got better” after enabling HTTP timing** partly because **rollout restart** reset pools / memory / event-loop state — stability signal, not proof the code was wrong.

**Prove it (second terminal while preflight or edge smoke runs):**
  kubectl top pods -n off-campus-housing-tracker
  kubectl top nodes
Watch: CPU or memory **> ~80%**, any pod **≥ ~1 core** sustained, Postgres- or Envoy-shaped spikes. Hooks also **warn** at 80% (K6_SUITE_WARN_*) and can **fail** at **85%** node CPU (K6_SUITE_NODE_CPU_MAX).

WHAT WE DO (orchestration + visibility — maps to “do this properly”)
--------------------------------------------------------------------
Implemented in **scripts/lib/k6-suite-resource-hooks.sh** and wired into **run-housing-k6-edge-smoke.sh**, **load/run-k6-phases.sh**, **run-all-test-suites.sh**, **run-preflight-scale-and-all-suites.sh**:

  1) **Cooldown after every k6 block** — default **K6_SUITE_COOLDOWN_SEC=15** (GC, pools, sockets).
  2) **Extra settle after CAR-style tests** — **K6_SUITE_CAR_EXTRA_SEC=20**; optional **K6_SUITE_RESTART_ENVOY_AFTER_CAR=1** (+ sleep, disruptive) or **K6_SUITE_STABILITY_AGGRESSIVE=1**.
  3) **Log kubectl top nodes + pods** after each block — **K6_SUITE_LOG_TOP=1**; optional **K6_SUITE_LOG_TOP_BEFORE=1**; append to **bench_logs/k6-suite-resources-*.log** via **K6_SUITE_RESOURCE_LOG** / **K6_SUITE_RESOURCE_LOG_AUTO** (preflight 7a enables AUTO).
  4) **Fail when node CPU too high** — **K6_SUITE_FAIL_ON_NODE_CPU=1**, **K6_SUITE_NODE_CPU_MAX=85**; hook **exit code 3**. Preflight **fails the run** on rc=3 after service grid / after full grid hook / after 7c in-cluster k6 (not only warn). Set **K6_SUITE_FAIL_ON_NODE_CPU=0** for noisy labs or missing metrics-server.

**Optional harsh lab (Colima VM):** **K6_SUITE_COLIMA_DROP_CACHES=1** — before each k6 block, **colima ssh** runs **sync** + **drop_caches** (same idea as manual sync; sudo inside VM). Not for production.

**Preflight-specific:** Sources hooks at start of **7a**; after **run-housing-k6-edge-smoke.sh** runs **k6_suite_after_k6_block preflight-after-k6-service-grid** (extra top + cooldown before Playwright).
