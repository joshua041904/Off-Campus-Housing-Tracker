feat(ai-control-plane,observability): add phased control-loop + canary arbitration scaffolding

Introduce Phase A/B/C control-plane foundations so analytics can move from passive
observability to guarded autonomous governance without unsafe flapping.

- Phase A (runtime + safe actuation):
  - Added in-process control-plane state/runtime for mode, active model, prompt version,
    canary percent, and arbitration weights.
  - Added controller loop (`off|observe|enforce`) with cooldown/hysteresis and
    deterministic decision logging before enforcement.
  - Wired runtime controls into listing generation path:
    - degraded mode token throttling
    - legacy mode short-circuit fallback
    - runtime-selected active model + prompt version envelope.
  - Added authenticated internal endpoints for control-plane state read/patch.

- Phase B (signal plane + Prometheus integration):
  - Added model/prompt/drift metrics:
    - analytics_model_{quality,reliability,latency,cost}_*
    - analytics_prompt_quality_score / analytics_prompt_requests_total
    - analytics_embedding_drift_score
  - Added control-plane rule pack with recordings/alerts for:
    - ai_model_health_score
    - analytics_embedding_drift_zscore
    - analytics_prompt_quality_delta_10m
    - embedding drift and prompt regression alerts.
  - Controller can now query Prometheus-derived composite/drift/prompt signals
    while retaining local fallback behavior.

- Phase C starter (canary + active-active hooks):
  - Added deterministic canary sampling and weighted arbitration helper.
  - Added shadow/canary arbitration instrumentation:
    - analytics_model_arbitration_runs_total
    - analytics_model_wins_total
    - analytics_model_arbitration_disagreement_ratio
  - Added cluster health gauge + Prometheus rule pack for
    ai_cluster_route_weight (health × inverse latency × inverse cost).
  - Added rollout runbook: docs/ai-control-plane-phase-c-runbook.md

- Observability plumbing:
  - Wired new rule ConfigMaps into kustomization and Prometheus projected mounts.
  - Added app-config defaults for control-plane mode, Prometheus queries,
    model cost book, canary model/shadow flags, and cluster id.

Net effect:
- deterministic, auditable control decisions in observe mode
- conservative autonomous actions when promoted to enforce mode
- canary/arbitration and multi-cluster routing signals ready for progressive rollout
