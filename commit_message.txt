feat(perf-lab): readiness gate, dashboards, operator matrix, CI fixture (PR #3)

WHY
---
  • Protocol scoring and capacity JSONs existed, but there was no single operator playbook,
    no automated gate, and no CI hook that could run without Colima/k6.
  • Dominance unlock had to stay consistent when JSON rounds τ to 0 while floats still show
    a tiny H3 gain.

WHAT CHANGED
------------
  1) scripts/protocol/declare-readiness.js
     • Multi-gate readiness: utilization @ pool 10, τ vs h3_transport_unlocked, fail_rate,
       tail_penalty vs 4× median, winner stability/composite consistency, max pool.
     • Optional: --strict-envelope, --panic-scan, --max-pool.

  2) scripts/protocol/compute-happiness.js
     • tail_penalty per protocol; winner_utilization_pool_10; flat jq aliases on matrix rows.
     • H3 unlock only if lambda3 > lambda2 * 1.0001 (material τ); note when not material.

  3) scripts/protocol/build-envelope-dashboard.js → envelope-dashboard.json
  4) scripts/protocol/build-dominance-heatmap.js → transport-dominance-heatmap.json
  5) scripts/protocol/validate-production-readiness.sh (+ --fixture)
  6) scripts/protocol/fixtures/perf-lab-pass/* — CI/passing smoke JSON
  7) docs/perf/PERFORMANCE_LAB_COMMAND_MATRIX.md — official command matrix
  8) .github/workflows/protocol-readiness.yml — fixture gate on scripts/protocol changes
  9) Makefile: perf-lab-dashboards, declare-readiness; capacity-one chains perf-lab-dashboards
 10) GITHUB_PR_DESCRIPTION_PR3.txt — PR body for paste
