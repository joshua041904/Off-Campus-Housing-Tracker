feat(perf): strict envelope deploy gate, adaptive pool suggest, H2 default hints (τ<0)

WHY
---
  • Operators needed production to fail fast when lab-recommended pools or stream caps exceed
    what is declared for k8s/ingress, without waiting for load regressions.
  • τ<0 services should not chase HTTP/3 as the default edge path; adaptive λ-based pool hints
    support a future controller without blocking on metrics plumbing today.

WHAT CHANGED
------------
  1) infra/k8s/base/config/strict-envelope.json
     • Declared configured_db_pool_max per service + ingress http2/http3 max concurrent streams.

  2) scripts/protocol/strict-envelope-check.js
     • Fails if recommended_pool > configured_db_pool_max or stream_cap > ingress cap.
     • Skips when capacity-recommendations.json is missing unless --require-lab.

  3) scripts/deploy-dev.sh
     • Runs strict-envelope-check before kustomize apply; SKIP_STRICT_ENVELOPE=1 to bypass.

  4) scripts/protocol/adaptive-pool-suggest.js
     • pool >= ceil(λ / (μ * util)) with default util=0.75; --observed-rps JSON input.

  5) scripts/protocol/build-transport-default-hints.js + Makefile transport-routing-hints*
     • transport-default-hints.json from protocol-happiness-matrix; --also-k8s writes
       infra/k8s/base/config/transport-routing-defaults.json.

  6) Makefile: capacity-one chains transport-routing-hints; strict-envelope-check;
     adaptive-pool-suggest (OBSERVED_RPS_JSON=…).

  7) scripts/protocol/fixtures: capacity-recommendations.json, strict-envelope-ci.json,
     example-observed-rps.json — CI + examples.

  8) .github/workflows/protocol-readiness.yml — strict-envelope job step + path filter.

  9) scripts/protocol/validate-production-readiness.sh — strict envelope phase (fixture uses CI envelope).

 10) docs/perf/PERFORMANCE_LAB_COMMAND_MATRIX.md — new rows + deploy/adaptive sections.
