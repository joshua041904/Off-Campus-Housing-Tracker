feat(och): E2E traffic stack + shared HTTP guard; fix(ci) listings /create + cron-jobs; Playwright + main baseline docs

WHY
---
  • After PR #15 (feature/system-build → main), listings-service failed CI with TS1472/TS1005: the
    POST /create handler was truncated — outer try/catch and app.post(...) never closed, and
    `return app` appeared inside the route. INSERT also referenced undefined `lat`/`lng`.
  • cron-jobs Docker/CI could hit `sh: tsc: not found` when `tsc` was not resolved from PATH;
    `pnpm exec tsc` pins the compiler to the package.
  • Parallel Playwright workers can stampede the gateway/backends; we shape load with a traffic
    shaper, optional Redis weighted budget, watchdog throttle, and a **labeled** E2E inflight cap
    (429 when over cap) — not by cutting workers to 2. Heavy spec files run in **serial** mode;
    Playwright stays at **4 workers** with **x-test-mode** + **x-e2e-test** on all browser requests.
  • UI E2E flakes tied to saturation and brittle locators: stable testids, section-scoped flows,
    HTTPS-only E2E base URL normalization, strict-edge script with dev CA.
  • Redis middleware typing: use ReturnType<typeof createClient> so gateway code matches the
    concrete client from createClient(), not a mismatched RedisClientType generic.
  • Integration branch `feature/system-build` is merged (PR #15); perf/docs should baseline on
    `main` and the remote feature branch was removed to avoid stale checkouts.
