perf(observability,load): listings diagnostics, k6 hooks, edge/gateway + suite pacing

First integrated PR: load-lab visibility, safer k6 defaults, preflight pacing, and edge fixes.

WHY
---
- Listings can look perfect under isolated k6 but degrade inside the full suite → cluster
  contention vs service bug. We add listings-side logs + between-test kubectl top/cooldowns.
- k6 should run without forgetting -e BASE_URL; stale checkouts should not hard-fail listings health.
- Colima QUIC capture and gateway JSON errors were causing false reds; housing HTTP checks were
  narrower than the HTTP/3 loop.

HOW TO RUN (minimal cheat sheet)
--------------------------------
Repo root = Off-Campus-Housing-Tracker.

  # Edge k6 grid (all services through Caddy hostname)
  SSL_CERT_FILE="$PWD/certs/dev-root.pem" ./scripts/run-housing-k6-edge-smoke.sh

  # Single listings health
  SSL_CERT_FILE="$PWD/certs/dev-root.pem" k6 run scripts/load/k6-listings-health.js

  # Big preflight, no pgbench (CPU gate off if metrics noisy)
  K6_SUITE_FAIL_ON_NODE_CPU=0 RUN_PGBENCH=0 ./scripts/run-preflight-scale-and-all-suites.sh

  # Listings diagnostics: build → load (Colima) → restart → env
  docker build -f services/listings-service/Dockerfile -t listings-service:dev .
  docker save listings-service:dev | colima ssh -- docker load
  kubectl rollout restart deployment/listings-service -n off-campus-housing-tracker
  kubectl set env deployment/listings-service -n off-campus-housing-tracker \
    LISTINGS_HTTP_TIMING=1 LISTINGS_HTTP_POOL_STATS_MS=2000
  kubectl logs -n off-campus-housing-tracker deployment/listings-service -c app --tail=200 | grep listings-http-timing

  # Perf report (when host can reach housing Postgres ports)
  ./scripts/perf/run-perf-full-report.sh
