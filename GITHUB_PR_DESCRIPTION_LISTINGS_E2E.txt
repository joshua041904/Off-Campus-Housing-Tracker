## Summary

This PR description covers the **entire webapp Playwright suite** (all edge E2E specs), not only listings. The suite exercises **guest shell**, **auth + booking flows**, **listings + maps/filters**, **analytics** (API + UI), and an **optional screenshot** job.

`webapp/playwright.config.ts` now defines **five named projects** so you can run or review by area; every `e2e/*.spec.ts` file belongs to exactly one project (no duplicate runs). Listings-focused work lives under project **`03-listings`**.

### Related: first / team PR narrative

Use **`GITHUB_PR_DESCRIPTION.txt`** for the **primary** long-form PR body: green-team bring-up (Colima, MetalLB, TLS, Ollama, DB restore), **three-reviewer split** (Joshua / Franco / Arkar), k6 and preflight orchestration, and the full testing map inside that document.

**This file (`GITHUB_PR_DESCRIPTION_LISTINGS_E2E.txt`)** is the **companion** for **browser E2E only**: the five Playwright projects, the **23** tests, copy-paste **Commands** below, and what reviewers should verify on the webapp + edge.

## PR dependency and merge order (important)

This is the **second PR** and it **depends on PR1** (`GITHUB_PR_DESCRIPTION.txt`) being applied first.

## What to focus on when reviewing

1. **Edge readiness** — Most tests need `GET /api/readyz` **200** (auth gRPC healthy). Guest-only cases may only need `/api/healthz` / reachable pages.
2. **03-listings** — Requires **webapp** + **listings-service** images aligned with the branch (search params, create body, proto/HTTP parity). Skips in specs often mean **stale webapp** on the cluster.
3. **02-auth-booking** — Touches **auth-service**, **booking-service**, **api-gateway** routing and JWT.
4. **04-analytics** — Listing-feel may hit **Ollama** or a stub; test allows either success or surfaced API error.
5. **Maps** — Embed iframe **or** placeholder is acceptable depending on whether `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` was baked at **docker build** time.

## How to run the full process

### Prerequisites

- Hostname resolves (e.g. `off-campus-housing.test` → cluster ingress).
- Trust dev CA: `certs/dev-root.pem` (used as `NODE_EXTRA_CA_CERTS`).
- **`E2E_API_BASE`**: HTTPS only; `http://127.0.0.1:4020` is rejected by config (normalized to default edge).

### Preflight / full pipeline (Playwright is step **7a8**)

**`./scripts/run-preflight-scale-and-all-suites.sh`** runs the housing Vitest + HTTP suites + k6 edge grid, then invokes **`./scripts/run-playwright-e2e-preflight.sh`** (unless `RUN_PREFLIGHT_PLAYWRIGHT=0`). That runs the **same 23-test / 5-project** Playwright suite as a standalone edge check. See the header comments in `scripts/run-preflight-scale-and-all-suites.sh` (search for `7a8` and `Playwright`).

## Commands (copy-paste)

### Webapp package (`cd webapp` **or** `pnpm --filter webapp run <script>`)

Runs use **`NODE_EXTRA_CA_CERTS`** via the script definitions in `webapp/package.json`.

```bash
# Per-project (Chromium, one project each)
pnpm run test:e2e:01-guest-shell
pnpm run test:e2e:02-auth-booking
pnpm run test:e2e:03-listings
pnpm run test:e2e:04-analytics

# Screenshots: sets E2E_SCREENSHOTS=1 and runs project 05 only
pnpm run test:e2e:05-screenshots

# Quick gate: guest shell + analytics (projects 01 + 04)
pnpm run test:e2e:smoke

# Entire suite (all projects — 23 listed, 22 executed by default)
pnpm run test:e2e
```

From repo root without changing directory:

```bash
pnpm --filter webapp run test:e2e:01-guest-shell
pnpm --filter webapp run test:e2e:02-auth-booking
pnpm --filter webapp run test:e2e:03-listings
pnpm --filter webapp run test:e2e:04-analytics
pnpm --filter webapp run test:e2e:05-screenshots
pnpm --filter webapp run test:e2e:smoke
pnpm --filter webapp run test:e2e
```

## Listings-specific coverage (project 03)

Stable `data-testid` hooks on `webapp/app/listings/page.tsx` (amenity filters, detail panel). Helper `firstListingIdFromSearch()` in `e2e/helpers.ts` for “load by ID”.

| Area | Asserts |
|------|---------|
| Guest CTA | Browse + search; **Log in** in `main`; no create form |
| Filters / sort / recency | Submit without `listings-api-error` |
| Amenities | Laundry + dishwasher (testid or label fallback) |
| Load by ID | Requires ≥1 listing in search index |
| Pet-friendly | Create + filter finds title |
| Geo / map | Coordinates + iframe **or** key placeholder |

**Note:** “Load listing by ID” skips on an empty index. Seed data or run create tests first.
