## 2. Executive summary

We hardened the **load lab** and **observability** story for **Off-Campus Housing (OCH)**:

| Theme | What we did | Why it matters |
|--------|-------------|----------------|
| **Listings diagnostics** | Env-driven HTTP timing + optional pg pool stats + startup log | Separates **in-process** vs **upstream** vs **DB pool** when k6 is bad in a **suite** but fine **alone**. |
| **k6 edge ergonomics** | Default `BASE_URL` + defensive fallback in listings health | Fewer missing `-e BASE_URL` failures. |
| **Suite orchestration** | After each k6 block: `kubectl top`, cooldowns, optional **node CPU% fail** | Surfaces **cluster contention**; less back-to-back k6 poisoning (Colima/k3s). |
| **Gateway / transport / suites** | `x-loadtest` on k6, gateway **400** on bad JSON, QUIC capture fallbacks, broader housing HTTP/2 checks | Fewer **false reds** on real edge paths. |
| **Perf reporting** | `scripts/perf/*` EXPLAIN + k6 report aggregation; **canonical 10-file bundle** + **k6 protocol matrix** | One zip for lab handoff; compare h1 / ALPN(h2) / h3 on gateway health. |

**Mental model:** *A service can be healthy while the **full preflight** is red because load stacks. This PR adds **visibility** (top, cooldowns, listings logs) so we do not confuse the two.*

**Long-form:** [`docs/perf/TAIL_LATENCY_AND_CROSS_SERVICE_ANALYSIS.md`](docs/perf/TAIL_LATENCY_AND_CROSS_SERVICE_ANALYSIS.md)

## 3. Prerequisites (one-time) — not macOS-only

You need a **Kubernetes** context and namespace **`off-campus-housing-tracker`** (created on the Colima path).

| Need | macOS (typical) | Linux (typical) | Windows |
|------|-----------------|-----------------|----------|
| **Container engine** | **Colima** (`brew install colima`) or Docker Desktop | Docker Engine + Compose plugin | **WSL2** + Docker / Colima in WSL |
| **kubectl** | `brew install kubectl` | [k8s install guide](https://kubernetes.io/docs/tasks/tools/) | WSL |
| **k6** | `brew install k6` | [k6 install](https://grafana.com/docs/k6/latest/set-up/install-k6/) | WSL / binary |
| **OpenSSL + keytool (JDK)** | `brew install openssl` + Temurin JDK | `openssl`, `openjdk-*-jdk` | WSL |
| **Node 20 + pnpm** | `brew install node`; `corepack enable` | nvm / distro packages | WSL |
| **PostgreSQL client** | `brew install libpq` | `postgresql-client` | WSL — **`pg_restore` 16.x** for restores |

**Always:** `certs/dev-root.pem` after TLS (**§4.6**). **`/etc/hosts`:** `off-campus-housing.test` → edge/LB IP — see [`docs/LOCAL_TLS_AND_TESTING_GUIDE.md`](docs/LOCAL_TLS_AND_TESTING_GUIDE.md). **metrics-server** (recommended) for `kubectl top` in hooks. **curl 8.19+** with HTTP/3 on **PATH** where needed (**§4.5**).

**k3d:** `REQUIRE_COLIMA=0` for preflight; skip **§4.2**; same **§4.3–4.9** pattern for infra + TLS + Ollama + cluster + gate.

## 4. First-time setup — from zero to preflight

*For people who have **never** brought this stack up (first-time setup). Same steps as [`docs/PR_SECOND_ONBOARDING.md`](docs/PR_SECOND_ONBOARDING.md) — keep both in sync.*

**Cross-reference map (§4):** **§4.2** Colima → **§4.3** Docker infra → **§4.4** what preflight does / does not → **§4.5** curl → **§4.6** TLS+JKS → **§4.7** Ollama → **§4.8** cluster+DB restore → **§4.9** gate+preflight → **§4.10** after preflight → **§4.11** one-paste block.

### 4.0 First-time fast path (copy/paste)

```bash
cd /path/to/Off-Campus-Housing-Tracker

# 1) Cluster + namespaces + MetalLB
./scripts/setup-new-colima-cluster.sh

# 2) External infra (Postgres, Redis, MinIO, Kafka/ZK)
RESTORE_BACKUP_DIR=latest ./scripts/bring-up-external-infra.sh

# 3) Cluster apps + restore path
RESTORE_BACKUP_DIR=latest ./scripts/bring-up-cluster-and-infra.sh

# 4) Optional but required for full analytics parity (LLM listing-feel)
# ollama serve (separate terminal), then wire OLLAMA_BASE_URL per §4.7

# 5) Full gate (preflight + suites; includes step 7a8 Playwright unless disabled)
RUN_PGBENCH=0 ./scripts/run-preflight-scale-and-all-suites.sh
```

If step 5 fails, go to **§10.1 Common error codes** first, then re-run this block.

### 4.1 Recommended order (checklist)

1. **Tools** — §3 table  
2. **Colima + namespaces + MetalLB** — `./scripts/setup-new-colima-cluster.sh` (**§4.2**)  
3. **TLS / Kafka JKS on disk** — **§4.6** before Kafka (or `SKIP_KAFKA=1` on first `bring-up-external-infra`)  
4. **Host Docker infra** — `./scripts/bring-up-external-infra.sh` (**§4.3**)  
5. **Ollama + model** — **§4.7** (first-time: **required** for full **analytics-service** LLM paths, **`k6-analytics-listing-feel.js`**, and Playwright analytics flows; wire **`OLLAMA_BASE_URL`** on the deployment **before** you treat preflight as full parity)
6. **Cluster + DB restore** — `RESTORE_BACKUP_DIR=latest ./scripts/bring-up-cluster-and-infra.sh` (**§4.8**)  
7. **curl HTTP/3 on PATH** — **§4.5**  
8. **Gate + preflight** — **§4.9**  
9. **More runs / perf** — **§4.10** · **§6** · **§7** · **`scripts/perf/*`** (**§6.8**)

**Ultra one-shot:** [`docs/RUN_PIPELINE_ORDER.md`](docs/RUN_PIPELINE_ORDER.md) · `setup-full-off-campus-housing-stack.sh` (from `setup-new-colima-cluster.sh`).

### 4.7 Ollama — first-time setup (full analytics / listing-feel)

**First-time setup:** treat **Ollama** as part of setup so **analytics-service** LLM routes, **`k6-analytics-listing-feel.js`** (edge smoke **§6.3**), and **webapp** analytics checks match what the rest of the team runs. The service **starts without** Ollama but returns **fallback copy** instead of real inference — that is **not** full parity ([`services/analytics-service/src/ollama.ts`](services/analytics-service/src/ollama.ts)).

**Install (macOS — Homebrew):**

```bash
brew install ollama
```

**Or** [ollama.com/download](https://ollama.com/download) (macOS / Linux / Windows).

**Run the daemon** (default **`127.0.0.1:11434`**):

```bash
ollama serve
```

**Pull the model** the code expects (default **`llama3.2`** — override with **`OLLAMA_MODEL`**):

```bash
ollama pull llama3.2
```

**Wire the cluster:** set **`OLLAMA_BASE_URL`** (and **`OLLAMA_MODEL`** if needed) on **`analytics-service`** **before** preflight if you want a green pass on all analytics paths:

- **Docker Desktop / Colima on Mac:** `OLLAMA_BASE_URL=http://host.docker.internal:11434`  
- **Linux:** host IP or e.g. `http://172.17.0.1:11434` (depends on Docker bridge)

**References:** [`services/analytics-service/README.md`](services/analytics-service/README.md) · **§6.3** (`k6-analytics-listing-feel.js`) · **§6.7** (Playwright analytics specs). If Ollama is intentionally down for a faster run: **`SKIP_K6_ANALYTICS_LISTING_FEEL=1`** (document that the run was partial).

## 10. Troubleshooting (FAQ)

| Issue | What to do |
|-------|------------|
| No `[listings-http-timing]` after `set env`? | Rebuild image, load into VM, rollout — env does not patch old binaries. |
| `BASE_URL must be https`? | Fix `k6-strict-edge-tls.js` or `-e BASE_URL=https://off-campus-housing.test` |
| Preflight slower? | Cooldowns by design; lower `K6_SUITE_COOLDOWN_SEC` locally only. |
| CPU gate failed? | `K6_SUITE_FAIL_ON_NODE_CPU=0` or raise `K6_SUITE_NODE_CPU_MAX`. |
| `kubectl top` errors? | Install metrics-server or disable CPU gate. |
| `k3s ctr import` failed on Colima? | `docker save ... \| colima ssh -- docker load` |
| Colima on Linux? | [Colima install](https://github.com/abiosoft/colima#installation) — then `setup-new-colima-cluster.sh` |
| Docker unreachable? | `colima start --with-kubernetes` or `docker context use colima` |

### 10.1 Common error codes (curl/x509/503)

| Error | Usually means | First thing to do |
|------|----------------|-------------------|
| `curl: (7)` | Connection refused / host not reachable / wrong target port | `colima status`; `kubectl get nodes`; verify hostname resolves; for Colima API tunnel run `./scripts/colima-forward-6443.sh` |
| `curl: (28)` | Timeout (service slow/not ready, edge not yet healthy) | Retry after readiness checks: `curl --cacert certs/dev-root.pem https://off-campus-housing.test/api/readyz` |
| `curl: (35)` | TLS handshake failure (protocol/cipher mismatch or broken cert chain) | Re-sync TLS secrets + CA (`pnpm run reissue`), then rerun strict TLS checks |
| `curl: (55)` | Send/recv reset mid-stream (often unstable edge/pod restart during request) | Check pods restarting (`kubectl get pods -n off-campus-housing-tracker`), rerun after rollout stabilizes |
| `curl: (60)` | Certificate verify failure (CA mismatch / stale certs) | Ensure `certs/dev-root.pem` is current; run `pnpm run reissue`; verify `dev-root-ca` + `service-tls` in cluster |
| `curl: (70)` | TLS client cert path/permissions or cert formatting issue | Recreate local certs (`./scripts/dev-generate-certs.sh`) and rerun preflight |
| `curl: (77)` | CA file unreadable/missing path | Check `certs/dev-root.pem` exists and is non-empty; verify `SSL_CERT_FILE` / `--cacert` path |
| `x509: certificate signed by unknown authority` | Client trust store missing current dev root CA | Sync trust from cluster (`ensure-strict-tls-mtls-preflight.sh` / preflight) and use `certs/dev-root.pem` |
| HTTP `503` with cert/TLS symptoms | Upstream not ready after cert rotation or secret mismatch | Wait for rollout status, verify `service-tls` + `dev-root-ca`, then re-run failing step |

Fast recovery sequence (first-time friendly):

```bash
./scripts/dev-generate-certs.sh
./scripts/ensure-housing-cluster-secrets.sh
pnpm run reissue
RUN_PGBENCH=0 ./scripts/run-preflight-scale-and-all-suites.sh
```
