## 2. Executive summary

We hardened the **load lab** and **observability** story for **Off-Campus Housing (OCH)**:

| Theme | What we did | Why it matters |
|--------|-------------|----------------|
| **Listings diagnostics** | Env-driven HTTP timing + optional pg pool stats + startup confirmation log | Separates **in-process** vs **upstream** vs **DB pool** when k6 is bad in a **suite** but fine **alone**. |
| **k6 edge ergonomics** | Default `BASE_URL` + defensive fallback in listings health | Fewer missing `-e BASE_URL` failures; tolerates stale `k6-strict-edge-tls.js` briefly. |
| **Suite orchestration** | After each k6 block: `kubectl top`, cooldowns, optional **node CPU% fail** | Surfaces **cluster contention**; reduces back-to-back k6 poisoning (Colima/k3s). |
| **Gateway / transport / suites** | `x-loadtest` on k6, gateway **400** on bad JSON, QUIC capture fallbacks, broader housing HTTP/2 checks | Fewer **false reds** on real edge paths. |
| **Perf reporting** | `scripts/perf/*` EXPLAIN + k6 report aggregation | One command for DB plans + load narrative. |

**Mental model:** *A service can be healthy while the **full preflight** is red because load stacks. This PR adds **visibility** (top, cooldowns, listings logs) so we do not confuse the two.*

**Team focus areas (long-form):** **`docs/perf/TAIL_LATENCY_AND_CROSS_SERVICE_ANALYSIS.md`** — (1) *Tail latency optimization (advanced)* — attribute p99/max via listings timing, pool stats, EXPLAIN, k6 thresholds; (2) *Cross-service performance analysis* — suite order effects, constant-arrival-rate contention, `kubectl top` during runs, and the **`K6_SUITE_*`** hooks that implement cooldowns + optional CPU / Envoy controls.

## 3. Prerequisites (one-time) — not macOS-only

Assume **`cd`** to repo root. You need a **working Kubernetes context** and namespace **`off-campus-housing-tracker`** (our scripts create it on Colima path).

| Need | macOS (typical) | Linux (typical) | Windows |
|------|-----------------|-----------------|---------|
| **Container engine** | **Colima** (`brew install colima`) or Docker Desktop | Docker Engine + Compose plugin (`docker.io`, `docker-compose-plugin`) | **WSL2** + Docker Desktop or Colima inside WSL — don’t run this stack from plain CMD without Docker |
| **kubectl** | `brew install kubectl` | Package manager or [install guide](https://kubernetes.io/docs/tasks/tools/) | Same via WSL |
| **k6** | `brew install k6` | [k6 install](https://grafana.com/docs/k6/latest/set-up/install-k6/) (deb/rpm/binary) | WSL / official binary |
| **OpenSSL + keytool (JDK)** | `brew install openssl`; Temurin JDK | `openssl`, `openjdk-*-jdk` packages | WSL |
| **Node 20 + pnpm** | `brew install node`; `corepack enable` / `npm i -g pnpm` | distro packages or nvm | WSL |
| **PostgreSQL client** | `brew install libpq` | `postgresql-client` | WSL — **`pg_restore` 16.x** recommended if restoring backups |

**Always:** **`certs/dev-root.pem`** after TLS bootstrap (§4.6). **`/etc/hosts`:** `off-campus-housing.test` → edge/LB IP (`docs/CERTS_AND_TESTING_FOR_MORTALS.md`). **metrics-server** in cluster (recommended) for `kubectl top` in k6 hooks. **curl 8.19+** with HTTP/3 on **PATH** where tests need it (§4.5 — system curl on macOS is often too old).

**k3d / Linux lab:** set **`REQUIRE_COLIMA=0`** for preflight when using k3d; you **skip** `setup-new-colima-cluster.sh` and follow your cluster create docs, then the same **external infra** + TLS + deploy path applies.

## 4. First timers — from zero to preflight (explicit script order)

*For teammates who have **never** brought this stack up. Nothing is “obvious”: Colima, MetalLB, eight Postgres databases, and Kafka SSL are all wired in scripts. **`docs/PR_SECOND_ONBOARDING.md`** mirrors this with more links — keep them in sync.*

### 4.1 Recommended order (checklist)

Do these **in order** on a clean machine (Colima path). Skip a step only if you know you already did it.

1. **Install tools** — §3 table (Docker, kubectl, k6, OpenSSL, JDK, Node/pnpm, `psql` if restoring).  
2. **New Colima cluster + namespaces + MetalLB** — **`./scripts/setup-new-colima-cluster.sh`** (§4.2).  
3. **TLS / Kafka JKS on disk** — §4.6 *before* starting Kafka (or use `SKIP_KAFKA=1` on first `bring-up-external-infra` run).  
4. **Host Docker infra** — **`./scripts/bring-up-external-infra.sh`** (Postgres ×8, Redis, Kafka, Zookeeper, MinIO, etc.) — §4.3.  
5. **Apps in cluster + DB restore (one-shot option)** — **`RESTORE_BACKUP_DIR=latest ./scripts/bring-up-cluster-and-infra.sh`** — §4.7 (or follow `setup-full-off-campus-housing-stack.sh` / `docs/RUN_PIPELINE_ORDER.md`).  
6. **curl HTTP/3 on PATH** — §4.5.  
7. **Gate + preflight** — §4.8.

**Ultra one-shot (if you trust the wrapper):** `setup-full-off-campus-housing-stack.sh` is mentioned from `setup-new-colima-cluster.sh`; see **`docs/RUN_PIPELINE_ORDER.md`** for how it relates to the steps above.

### 4.2 New Colima + k3s + namespaces + **MetalLB** — `setup-new-colima-cluster.sh`

Use when you **don’t** already have Colima running with k3s and the right namespaces.

```bash
cd /path/to/Off-Campus-Housing-Tracker
chmod +x scripts/setup-new-colima-cluster.sh
./scripts/setup-new-colima-cluster.sh
```

- Starts **Colima** with k3s, **`--network-address`**, default CPU/RAM/disk (override `CPU`, `MEMORY`, `DISK`).  
- Creates **`ingress-nginx`**, **`envoy-test`**, **`off-campus-housing-tracker`**.  
- Installs **MetalLB** via **`install-metallb-colima.sh`** unless **`SKIP_METALLB=1`**. Pool: unset = auto-detect; override e.g. **`METALLB_POOL=192.168.64.240-192.168.64.250`**.  
- If **`kubectl`** fails: `export KUBECONFIG=$HOME/.colima/default/kubeconfig` (script warns when kubeconfig may need merging).

**Then** run **§4.3** (external infra). Script footer also points at **`bring-up-external-infra.sh`** → build/load images → deploy.

### 4.3 Host-side Docker infra — `bring-up-external-infra.sh`

Brings up **Zookeeper, Kafka (strict TLS), Redis, MinIO, eight Postgres** services expected at **`host.docker.internal:5441`–`5448`** (and Kafka **`29094`**, etc.). **Required** before pods can talk to “external” databases.

```bash
cd /path/to/Off-Campus-Housing-Tracker
export PGPASSWORD=postgres   # or your compose password

# Optional: restore from newest backup set after Postgres healthy
RESTORE_BACKUP_DIR=latest ./scripts/bring-up-external-infra.sh
```

- **Kafka** needs **`certs/kafka-ssl/`** (e.g. `kafka.keystore.jks`) or the script will prompt / you set **`SKIP_KAFKA=1`** until §4.6 is done.  
- **`RESTORE_BACKUP_DIR`**: `latest` or a path under **`backups/`** — see script header and **`backups/README.md`**.  
- With Colima running, the script tries **`docker context use colima`** / **`DOCKER_HOST`** so **`docker compose`** hits the VM.

### 4.4 What `run-preflight-scale-and-all-suites.sh` already does (certs)

You don’t hand-paste PEMs every run. Preflight runs **`ensure-housing-cluster-secrets`**, **`ensure-strict-tls-mtls-preflight`**, and by default **`PREFLIGHT_REISSUE_CA=0`** when `service-tls` + `dev-root-ca` already exist.

**You still need** **`certs/kafka-ssl/`** on disk **before** Docker **Kafka** starts in §4.3 — `bring-up-external-infra.sh` enforces this. Do **§4.6** before or between cluster bring-up and Kafka.

### 4.5 curl **8.19+** + HTTP/3 — macOS **and** Linux

**macOS (Homebrew — Apple `/usr/bin/curl` usually has no `--http3`):**

```bash
brew install curl
export PATH="/opt/homebrew/opt/curl/bin:$PATH"   # Intel: /usr/local/opt/curl/bin
hash -r
curl -V
./scripts/verify-curl-http3.sh
```

Put **`export PATH=...`** in **`~/.zshrc`** / **`~/.bashrc`**. **`ensure-ready-for-preflight.sh`** nudges Homebrew curl when present.

**Linux:** install a **new enough** curl from your distro backports, [curl.se](https://curl.se/download.html), or static build; ensure **`curl --help all`** lists **`http3`** and **`./scripts/verify-curl-http3.sh`** passes.

### 4.6 TLS / mTLS / Kafka JKS (first time or clean machine)

Order matters — full detail: **`docs/CERT_GENERATION_STRICT_TLS_MTLS.md`**.

```bash
cd /path/to/Off-Campus-Housing-Tracker

# 1) CA + leaf + cluster secrets (KAFKA_SSL=1 keeps dev-root.key for next steps)
KAFKA_SSL=1 ./scripts/reissue-ca-and-leaf-load-all-services.sh

# 2) Envoy client cert → cluster
./scripts/generate-envoy-client-cert.sh
./scripts/strict-tls-bootstrap.sh

# 3) Kafka broker + client + JKS → certs/kafka-ssl/ + kafka-ssl-secret
./scripts/kafka-ssl-from-dev-root.sh
```

- **macOS browser trust:** `./scripts/lib/trust-dev-root-ca-macos.sh certs/dev-root.pem`  
- **Linux:** import **`certs/dev-root.pem`** into your browser / NSS store, or follow **`docs/CERTS_AND_TESTING_FOR_MORTALS.md`**.

### 4.7 One-shot: cluster + MetalLB path + **latest DB backup** — `bring-up-cluster-and-infra.sh`

This orchestrates **Kubernetes** bring-up / sync and often assumes **Docker infra** is already up (§4.3). From repo root:

```bash
export PGPASSWORD=postgres
RESTORE_BACKUP_DIR=latest ./scripts/bring-up-cluster-and-infra.sh
```

- **`latest`** = newest **`backups/all-8-*`** or **`all-7-*`** (see **`backups/README.md`**).  
- **Cluster already up?** `RESTORE_BACKUP_DIR=latest SKIP_CLUSTER=1 ./scripts/bring-up-cluster-and-infra.sh`  
- **Backup yourself:** `PGPASSWORD=postgres ./scripts/backup-all-dbs.sh`  
- Pods use **`host.docker.internal:5441`–`5448`**; if pods can’t reach Postgres, see **`scripts/colima-apply-host-aliases.sh`** and Colima/k3s networking docs.

**Preflight with MetalLB:** for Colima + LoadBalancer Caddy, use **`METALLB_ENABLED=1`** (§4.8).

### 6.4 Listings HTTP diagnostics in Kubernetes

1. Build:

```bash
docker build -f services/listings-service/Dockerfile -t listings-service:dev .
```

2. Load (Colima pattern when `ctr` import fails):

```bash
docker save listings-service:dev | colima ssh -- docker load
```

3. Rollout:

```bash
kubectl rollout restart deployment/listings-service -n off-campus-housing-tracker
kubectl rollout status deployment/listings-service -n off-campus-housing-tracker --timeout=180s
```

4. Enable:

```bash
kubectl set env deployment/listings-service -n off-campus-housing-tracker \
  LISTINGS_HTTP_TIMING=1 \
  LISTINGS_HTTP_POOL_STATS_MS=2000
kubectl rollout status deployment/listings-service -n off-campus-housing-tracker --timeout=180s
```

5. Confirm:

```bash
kubectl exec -n off-campus-housing-tracker deploy/listings-service -c app -- printenv LISTINGS_HTTP_TIMING
kubectl logs -n off-campus-housing-tracker deployment/listings-service -c app --tail=200 | grep listings-http-timing
```

Log every request:

```bash
kubectl set env deployment/listings-service -n off-campus-housing-tracker \
  LISTINGS_HTTP_TIMING=1 LISTINGS_HTTP_TIMING_MIN_MS=0
```

More detail: `docs/perf/LISTINGS_HTTP_TIMING.md`

### 6.6 Suite hooks tuning

Defaults: **15s** cooldown, **+20s** after constant-arrival-rate blocks, **85%** node CPU fail.

Friendly:

```bash
export K6_SUITE_FAIL_ON_NODE_CPU=0
export K6_SUITE_LOG_TOP=1
```

Faster local iteration (less realistic):

```bash
export K6_SUITE_COOLDOWN_SEC=3
export K6_SUITE_CAR_EXTRA_SEC=5
```

Optional Envoy restart after CAR tests:

```bash
export K6_SUITE_RESTART_ENVOY_AFTER_CAR=1
```

## 8. Environment variables reference

### Listings (pod)

| Variable | Example | Meaning |
|----------|---------|---------|
| `LISTINGS_HTTP_TIMING` | `1` | Enable diagnostics |
| `LISTINGS_HTTP_TIMING_MIN_MS` | `1000` / `0` | Min ms to log; `0` = all |
| `LISTINGS_HTTP_POOL_STATS_MS` | `2000` | Pool log interval; `0` = off |
| `LISTINGS_HTTP_SEARCH_DB_MIN_MS` | `50` | Log search DB phase threshold |

### k6 suite hooks

| Variable | Default | Meaning |
|----------|---------|---------|
| `K6_SUITE_COOLDOWN_SEC` | `15` | Sleep after each k6 block |
| `K6_SUITE_CAR_EXTRA_SEC` | `20` | Extra after CAR-style tests |
| `K6_SUITE_LOG_TOP` | `1` | Print `kubectl top` |
| `K6_SUITE_FAIL_ON_NODE_CPU` | `1` | Fail if node CPU% too high |
| `K6_SUITE_NODE_CPU_MAX` | `85` | Threshold |
| `K6_SUITE_TOP_NS` | `off-campus-housing-tracker` | Pod top namespace |

## 9. Troubleshooting (FAQ)

**No `[listings-http-timing]` after set env?** Rebuild image, load into VM, rollout — env does not patch old binaries. Look for startup line `enabled minMs=...`.

**`BASE_URL must be https`?** Update `k6-strict-edge-tls.js` or pass `-e BASE_URL=https://off-campus-housing.test`.

**Preflight slower?** Cooldowns by design; lower `K6_SUITE_COOLDOWN_SEC` locally only.

**CPU gate failed?** `K6_SUITE_FAIL_ON_NODE_CPU=0` or raise `K6_SUITE_NODE_CPU_MAX`.

**`kubectl top` errors?** Install metrics-server or ignore CPU gate.

**`k3s ctr import` failed on Colima?** Use `docker save ... | colima ssh -- docker load`.

**Colima on Linux?** Install via [Colima install docs](https://github.com/abiosoft/colima#installation) (brew, script, or package); then **`./scripts/setup-new-colima-cluster.sh`** same as macOS.

**Docker daemon unreachable?** Start Colima (`colima start --with-kubernetes`) or **`docker context use colima`** — see **`bring-up-external-infra.sh`** header.
