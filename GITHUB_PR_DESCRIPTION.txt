## 2. Executive summary

We hardened the **load lab** and **observability** story for **Off-Campus Housing (OCH)**:

| Theme | What we did | Why it matters |
|--------|-------------|----------------|
| **Listings diagnostics** | Env-driven HTTP timing + optional pg pool stats + startup log | Separates **in-process** vs **upstream** vs **DB pool** when k6 is bad in a **suite** but fine **alone**. |
| **k6 edge ergonomics** | Default `BASE_URL` + defensive fallback in listings health | Fewer missing `-e BASE_URL` failures. |
| **Suite orchestration** | After each k6 block: `kubectl top`, cooldowns, optional **node CPU% fail** | Surfaces **cluster contention**; less back-to-back k6 poisoning (Colima/k3s). |
| **Gateway / transport / suites** | `x-loadtest` on k6, gateway **400** on bad JSON, QUIC capture fallbacks, broader housing HTTP/2 checks | Fewer **false reds** on real edge paths. |
| **Perf reporting** | `scripts/perf/*` EXPLAIN + k6 report aggregation | One command for DB plans + load narrative. |

**Mental model:** *A service can be healthy while the **full preflight** is red because load stacks. This PR adds **visibility** (top, cooldowns, listings logs) so we do not confuse the two.*

**Long-form:** [`docs/perf/TAIL_LATENCY_AND_CROSS_SERVICE_ANALYSIS.md`](docs/perf/TAIL_LATENCY_AND_CROSS_SERVICE_ANALYSIS.md)

## 3. Prerequisites (one-time) — not macOS-only

You need a **Kubernetes** context and namespace **`off-campus-housing-tracker`** (created on the Colima path).

| Need | macOS (typical) | Linux (typical) | Windows |
|------|-----------------|-----------------|----------|
| **Container engine** | **Colima** (`brew install colima`) or Docker Desktop | Docker Engine + Compose plugin | **WSL2** + Docker / Colima in WSL |
| **kubectl** | `brew install kubectl` | [k8s install guide](https://kubernetes.io/docs/tasks/tools/) | WSL |
| **k6** | `brew install k6` | [k6 install](https://grafana.com/docs/k6/latest/set-up/install-k6/) | WSL / binary |
| **OpenSSL + keytool (JDK)** | `brew install openssl` + Temurin JDK | `openssl`, `openjdk-*-jdk` | WSL |
| **Node 20 + pnpm** | `brew install node`; `corepack enable` | nvm / distro packages | WSL |
| **PostgreSQL client** | `brew install libpq` | `postgresql-client` | WSL — **`pg_restore` 16.x** for restores |

**Always:** `certs/dev-root.pem` after TLS (**§4.6**). **`/etc/hosts`:** `off-campus-housing.test` → edge/LB IP — see [`docs/LOCAL_TLS_AND_TESTING_GUIDE.md`](docs/LOCAL_TLS_AND_TESTING_GUIDE.md). **metrics-server** (recommended) for `kubectl top` in hooks. **curl 8.19+** with HTTP/3 on **PATH** where needed (**§4.5**).

**k3d:** `REQUIRE_COLIMA=0` for preflight; skip **§4.2**; same **§4.3–4.9** pattern for infra + TLS + Ollama + cluster + gate.

### 4.7 Ollama — green team (full analytics / listing-feel)

**Green team:** treat **Ollama** as part of **first-time** setup so **analytics-service** LLM routes, **`k6-analytics-listing-feel.js`** (edge smoke **§6.3**), and **webapp** analytics checks match what the rest of the team runs. The service **starts without** Ollama but returns **fallback copy** instead of real inference — that is **not** full parity ([`services/analytics-service/src/ollama.ts`](services/analytics-service/src/ollama.ts)).

**Install (macOS — Homebrew):**

```bash
brew install ollama
```

**Or** [ollama.com/download](https://ollama.com/download) (macOS / Linux / Windows).

**Run the daemon** (default **`127.0.0.1:11434`**):

```bash
ollama serve
```

**Pull the model** the code expects (default **`llama3.2`** — override with **`OLLAMA_MODEL`**):

```bash
ollama pull llama3.2
```

**Wire the cluster:** set **`OLLAMA_BASE_URL`** (and **`OLLAMA_MODEL`** if needed) on **`analytics-service`** **before** preflight if you want a green pass on all analytics paths:

- **Docker Desktop / Colima on Mac:** `OLLAMA_BASE_URL=http://host.docker.internal:11434`  
- **Linux:** host IP or e.g. `http://172.17.0.1:11434` (depends on Docker bridge)

**References:** [`services/analytics-service/README.md`](services/analytics-service/README.md) · **§6.3** (`k6-analytics-listing-feel.js`) · **§6.7** (Playwright analytics specs). If Ollama is intentionally down for a faster run: **`SKIP_K6_ANALYTICS_LISTING_FEEL=1`** (document that the run was partial).

## 10. Troubleshooting (FAQ)

| Issue | What to do |
|-------|------------|
| No `[listings-http-timing]` after `set env`? | Rebuild image, load into VM, rollout — env does not patch old binaries. |
| `BASE_URL must be https`? | Fix `k6-strict-edge-tls.js` or `-e BASE_URL=https://off-campus-housing.test` |
| Preflight slower? | Cooldowns by design; lower `K6_SUITE_COOLDOWN_SEC` locally only. |
| CPU gate failed? | `K6_SUITE_FAIL_ON_NODE_CPU=0` or raise `K6_SUITE_NODE_CPU_MAX`. |
| `kubectl top` errors? | Install metrics-server or disable CPU gate. |
| `k3s ctr import` failed on Colima? | `docker save ... \| colima ssh -- docker load` |
| Colima on Linux? | [Colima install](https://github.com/abiosoft/colima#installation) — then `setup-new-colima-cluster.sh` |
| Docker unreachable? | `colima start --with-kubernetes` or `docker context use colima` |
