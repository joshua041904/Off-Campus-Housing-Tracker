## 2. Executive summary

We hardened the **load lab** and **observability** story for **Off-Campus Housing (OCH)**:

| Theme | What we did | Why it matters |
|--------|-------------|----------------|
| **Listings diagnostics** | Env-driven HTTP timing + optional pg pool stats + startup confirmation log | Separates **in-process** vs **upstream** vs **DB pool** when k6 is bad in a **suite** but fine **alone**. |
| **k6 edge ergonomics** | Default `BASE_URL` + defensive fallback in listings health | Fewer missing `-e BASE_URL` failures; tolerates stale `k6-strict-edge-tls.js` briefly. |
| **Suite orchestration** | After each k6 block: `kubectl top`, cooldowns, optional **node CPU% fail** | Surfaces **cluster contention**; reduces back-to-back k6 poisoning (Colima/k3s). |
| **Gateway / transport / suites** | `x-loadtest` on k6, gateway **400** on bad JSON, QUIC capture fallbacks, broader housing HTTP/2 checks | Fewer **false reds** on real edge paths. |
| **Perf reporting** | `scripts/perf/*` EXPLAIN + k6 report aggregation | One command for DB plans + load narrative. |

**Mental model:** *A service can be healthy while the **full preflight** is red because load stacks. This PR adds **visibility** (top, cooldowns, listings logs) so we do not confuse the two.*

## 3. Prerequisites (one-time)

- Kubernetes context (e.g. **Colima + k3s** or **k3d**), namespace **`off-campus-housing-tracker`**.  
- **Docker**, **kubectl**, **k6** (`brew install k6` on macOS).  
- **`certs/dev-root.pem`** at repo root.  
- **`/etc/hosts`:** `off-campus-housing.test` → edge IP.  
- **metrics-server** (recommended): enables `kubectl top` in hooks; without it, CPU check is skipped with a warning.

## 5. How to run — copy/paste recipes

### 5.1 k6 edge smoke grid

```bash
cd /path/to/Off-Campus-Housing-Tracker
SSL_CERT_FILE="$PWD/certs/dev-root.pem" ./scripts/run-housing-k6-edge-smoke.sh
```

Strict:

```bash
K6_GRID_STRICT=1 SSL_CERT_FILE="$PWD/certs/dev-root.pem" ./scripts/run-housing-k6-edge-smoke.sh
```

Skip optional:

```bash
SKIP_K6_ANALYTICS_LISTING_FEEL=1 SKIP_K6_BOOKING_SEARCH=1 \
  SSL_CERT_FILE="$PWD/certs/dev-root.pem" ./scripts/run-housing-k6-edge-smoke.sh
```

**Note:** Expect **cooldowns** and **`kubectl top`** between scripts (`scripts/lib/k6-suite-resource-hooks.sh`).

### 5.2 k6 listings health only

```bash
cd /path/to/Off-Campus-Housing-Tracker
SSL_CERT_FILE="$PWD/certs/dev-root.pem" k6 run scripts/load/k6-listings-health.js
```

### 5.3 Preflight + suites (main orchestrator)

See header of `scripts/run-preflight-scale-and-all-suites.sh` for full flags.

```bash
RUN_PGBENCH=0 ./scripts/run-preflight-scale-and-all-suites.sh
```

Example MetalLB:

```bash

```

```bash

```

Skip k6 grid inside preflight:

```bash

```

### 5.4 Listings HTTP diagnostics in Kubernetes

1. Build:

```bash
docker build -f services/listings-service/Dockerfile -t listings-service:dev .
```

2. Load (Colima pattern when `ctr` import fails):

```bash
docker save listings-service:dev | colima ssh -- docker load
```

3. Rollout:

```bash
kubectl rollout restart deployment/listings-service -n off-campus-housing-tracker
kubectl rollout status deployment/listings-service -n off-campus-housing-tracker --timeout=180s
```

4. Enable:

```bash
kubectl set env deployment/listings-service -n off-campus-housing-tracker \
  LISTINGS_HTTP_TIMING=1 \
  LISTINGS_HTTP_POOL_STATS_MS=2000
kubectl rollout status deployment/listings-service -n off-campus-housing-tracker --timeout=180s
```

5. Confirm:

```bash
kubectl exec -n off-campus-housing-tracker deploy/listings-service -c app -- printenv LISTINGS_HTTP_TIMING
kubectl logs -n off-campus-housing-tracker deployment/listings-service -c app --tail=200 | grep listings-http-timing
```

Log every request:

```bash
kubectl set env deployment/listings-service -n off-campus-housing-tracker \
  LISTINGS_HTTP_TIMING=1 LISTINGS_HTTP_TIMING_MIN_MS=0
```

More detail: `docs/perf/LISTINGS_HTTP_TIMING.md`

### 5.5 Perf report (EXPLAIN + k6)

```bash
./scripts/perf/run-perf-full-report.sh
```

À la carte: `./scripts/perf/run-all-explain.sh`, `./scripts/perf/run-all-k6-load-report.sh`  
Output: `bench_logs/perf-report-*/`

### 5.6 Suite hooks tuning

Defaults: **15s** cooldown, **+20s** after constant-arrival-rate blocks, **85%** node CPU fail.

Friendly:

```bash
export K6_SUITE_FAIL_ON_NODE_CPU=0
export K6_SUITE_LOG_TOP=1
```

Faster local iteration (less realistic):

```bash
export K6_SUITE_COOLDOWN_SEC=3
export K6_SUITE_CAR_EXTRA_SEC=5
```

Optional Envoy restart after CAR tests:

```bash
export K6_SUITE_RESTART_ENVOY_AFTER_CAR=1
```

## 7. Environment variables reference

### Listings (pod)

| Variable | Example | Meaning |
|----------|---------|---------|
| `LISTINGS_HTTP_TIMING` | `1` | Enable diagnostics |
| `LISTINGS_HTTP_TIMING_MIN_MS` | `1000` / `0` | Min ms to log; `0` = all |
| `LISTINGS_HTTP_POOL_STATS_MS` | `2000` | Pool log interval; `0` = off |
| `LISTINGS_HTTP_SEARCH_DB_MIN_MS` | `50` | Log search DB phase threshold |

### k6 suite hooks

| Variable | Default | Meaning |
|----------|---------|---------|
| `K6_SUITE_COOLDOWN_SEC` | `15` | Sleep after each k6 block |
| `K6_SUITE_CAR_EXTRA_SEC` | `20` | Extra after CAR-style tests |
| `K6_SUITE_LOG_TOP` | `1` | Print `kubectl top` |
| `K6_SUITE_FAIL_ON_NODE_CPU` | `1` | Fail if node CPU% too high |
| `K6_SUITE_NODE_CPU_MAX` | `85` | Threshold |
| `K6_SUITE_TOP_NS` | `off-campus-housing-tracker` | Pod top namespace |
| `K6_SUITE_TOP_ENVOY_NS` | `envoy-test` | Second namespace (empty = skip) |
| `K6_SUITE_RESTART_ENVOY_AFTER_CAR` | `0` | `1` = rollout envoy after CAR |

## 8. Troubleshooting (FAQ)

**No `[listings-http-timing]` after set env?** Rebuild image, load into VM, rollout — env does not patch old binaries. Look for startup line `enabled minMs=...`.

**`BASE_URL must be https`?** Update `k6-strict-edge-tls.js` or pass `-e BASE_URL=https://off-campus-housing.test`.

**Preflight slower?** Cooldowns by design; lower `K6_SUITE_COOLDOWN_SEC` locally only.

**CPU gate failed?** `K6_SUITE_FAIL_ON_NODE_CPU=0` or raise `K6_SUITE_NODE_CPU_MAX`.

**`kubectl top` errors?** Install metrics-server or ignore CPU gate.

**`k3s ctr import` failed on Colima?** Use `docker save ... | colima ssh -- docker load`.
