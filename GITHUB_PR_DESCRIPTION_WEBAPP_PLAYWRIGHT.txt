## 2. What you must do after merge (deploy checklist)

**Do not skip if you want the live edge to match this branch.**

1. **Listings service**  
   - Rebuild `services/listings-service` image.  
   - `kubectl rollout restart deployment/listings-service -n off-campus-housing-tracker`  
   - Ensure listings DB has `latitude` / `longitude` (see `infra/db/01-listings-schema-and-tuning.sql`).

2. **Webapp**  
   - Rebuild image from repo root so Next inlines public env at build time:  
     `docker build -f webapp/Dockerfile -t webapp:dev --build-arg NEXT_PUBLIC_GOOGLE_MAPS_API_KEY="YOUR_KEY" .`  
   - Rollout `deployment/webapp` in `off-campus-housing-tracker`.  
   - If you rely on runtime Secret only, you still need a build that includes the key **or** a build pipeline that passes the same `ARG` — see `docs/WEBAPP_GOOGLE_MAPS_AND_DEPLOY.txt`.

3. **Playwright**  
   - Run `./scripts/run-playwright-e2e-preflight.sh` against `https://off-campus-housing.test` (or `E2E_API_BASE`).  
   - Until the **new** webapp image is live, some tests **skip** when new `data-testid`s or lat/lng placeholders are missing (see `e2e/listings-filters-maps.spec.ts`).

## 3. Google Maps setup (operator steps)

Full step-by-step: **`docs/WEBAPP_GOOGLE_MAPS_AND_DEPLOY.txt`** (Console → Enable **Maps Embed API** → key → **HTTP referrer** restrictions → optional **API** restriction to Embed only).

**Referrers to allow (typical):**  
`https://off-campus-housing.test/*` and any wildcard subdomain you use for the edge.

**Important:** `NEXT_PUBLIC_*` is embedded in the client bundle at **`next build`** time. Passing the key only as a pod env var **without** rebuilding the image will **not** update an already-built bundle — use `--build-arg` as in section 2, or bake via CI.
