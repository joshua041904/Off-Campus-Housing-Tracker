# PR #4 — Deterministic CI contract: OCH matrices, certification pipeline, no phantom cluster

**Standalone PR:** Use **only** this file as the GitHub/GitLab description for **PR #4**. It is self-contained relative to **PR #2** (protocol happiness / dominance) and **PR #3** (readiness gates, dashboards, fixture CI).

**Title (suggested):** `ci(och): och-ci workflow, certification script, H2 anomaly gate, docker port + QUIC invariants; remove local-stack workflow`

**Stack position:** PR #1 capacity · PR #2 protocol happiness · PR #3 `declare-readiness` + dashboards + fixture CI · **PR #4 (this)** locks **layer separation**: docker-compose (infra), k3s (runtime), GitHub CI (structural only), local **`make certify`** (full cluster path). **Follow-up PRs** can apply pool / H2 tuning from lab data without re-litigating CI shape.

## Executive summary

This PR **stops improvising** across three realities (Compose, k3s, `ubuntu-latest`) and **writes down** a deterministic contract:

| Layer | Responsibility |
|--------|----------------|
| **docker-compose** | Infra only (Postgres 5441–5448, Redis 6380, Kafka, MinIO, …) |
| **k3s + MetalLB + Caddy** | Application runtime and HTTP/3 edge |
| **GitHub CI** | Build/test matrix, static protocol anomaly gate, Python transport tooling smoke, QUIC hostname static check — **no assumption of a private cluster** |
| **Local certification** | `make certify` / `scripts/ci/run-full-certification.sh` — full slice including Playwright strict, `transport-lab`, `declare-readiness --strict-quic` when run on a machine with Docker + cluster |

**Highlights**

- **`och-ci`** (`.github/workflows/ci.yml`): ordered jobs — **protocol-anomaly → build → test → transport-validation → quic-hostname-invariant**; **Strict Playwright** only when repo variable **`RUN_STRICT_PLAYWRIGHT == 'true'`** (self-hosted / reachable edge).
- **HTTP/2 collapse** is a **first-class CI failure** when `bench_logs/performance-lab/protocol-matrix-anomalies.json` has `"any_http2_collapse_anomaly": true` (regenerated from CSV fallback via `EXTRACT_PROTOCOL_MATRIX_FROM_CSV=1`). That is **enforcement**, not a broken pipeline — fix tuning, then green.
- **Removed** `.github/workflows/local-integration.yml` (docker-compose “app simulation” on Actions). Cluster-first posture; local compose flows stay documented for manual use.
- **`pnpm/action-setup@v4` (9.11.0) before `setup-node`** everywhere **`pnpm` runs** — fixes “Unable to locate executable file: pnpm”.
- **`scripts/ci/run-full-certification.sh`**: canonical local certification; **`CI=true`** exits after unit tests (no transport-lab / declare-readiness on runner); **non-CI** requires **`docker`** on PATH before full path.
- **`scripts/ci/verify-docker-ports.sh`** + **`make verify-docker-ports`** — invariant **5441–5448** and **6380** visible in `docker ps` (IPv4 / IPv6 mapping patterns).
- **`scripts/ci/verify-quic-hostname-invariant.sh`** — static guard: no raw **`https://<IP>`** defaults for **`BASE_URL`** under **`scripts/load/**/*.js`** (SNI / QUIC contract).
- **`scripts/perf/extract-protocol-matrix.js`**: **`EXTRACT_PROTOCOL_MATRIX_FROM_CSV=1`** rebuilds rows + **`protocol-matrix-anomalies.json`** when `bench_logs/run-*/protocol-matrix/` is missing (flattened runs).
- **`scripts/protocol/declare-readiness.js`**: reads **`protocol-matrix-anomalies.json`** under `--perf-dir` when transport artifact omits **`protocol_integrity`** (e.g. `protocol_integrity: null`), so matrix collapse matches extraction output.
- **`.github/workflows/docker-build.yml`**: **`och-docker-build`** — image build matrix for real OCH services + **webapp** (no legacy RP services; **`python-ai-service`** omitted until a Dockerfile exists in-tree).
- **`webapp`**: strict verticals + combined strict run set **`PLAYWRIGHT_STRICT_HTTP3=1`**; **`system-integrity.spec.ts`** asserts **cross-service state** (listings count, trust score, notification toggles), not only HTTP 200s.
- **`docs/LOCAL_DEV.md`**: CI section updated — no longer references deleted local-integration workflow.

## What changed (by area)

### A. `.github/workflows/ci.yml` (`och-ci`)

1. **`protocol-anomaly`** (runs first): `EXTRACT_PROTOCOL_MATRIX_FROM_CSV=1 node scripts/perf/extract-protocol-matrix.js`, assert anomalies file exists, **`grep '"any_http2_collapse_anomaly": false'`** (hard fail on collapse).
2. **`build`** matrix: api-gateway, auth, listings, booking, messaging, trust, analytics, media, notification, **cron-jobs** — `pnpm -C services/<svc> build`.
3. **`test`** matrix: same services **except cron-jobs** — `pnpm -C services/<svc> test --if-present`.
4. **`transport-validation`**: Python file existence, **`py_compile`**, **`transport_validator.py`** with no args → exit **2** + **`no pcap provided`** (tooling contract).
5. **`quic-hostname-invariant`**: `bash scripts/ci/verify-quic-hostname-invariant.sh`.
6. **`playwright-strict`**: `if: vars.RUN_STRICT_PLAYWRIGHT == 'true'`, `PLAYWRIGHT_STRICT_HTTP3: "1"`, Playwright browser install + strict verticals + integrity.

**Dependency chain:** `protocol-anomaly` → `build` → `test` → `transport-validation` → `quic-hostname-invariant`; optional Playwright after `transport-validation` when variable set.

### B. `.github/workflows/docker-build.yml` (`och-docker-build`)

- Matrix build for OCH services + **webapp**; **`file:`** expression chooses **`webapp/Dockerfile`** vs **`services/<name>/Dockerfile`**.

### C. Removed

- **`.github/workflows/local-integration.yml`** — removed to avoid implying GitHub-hosted “full local stack” as the architecture.

### D. `.github/workflows/protocol-validation.yml`

- Path filters extended (**`scripts/ci/**`**, **`scripts/lib/**`**, **`scripts/perf/extract-protocol-matrix.js`**).
- **ShellCheck** for **`scripts/ci/run-full-certification.sh`**, **`verify-docker-ports.sh`**, **`verify-quic-hostname-invariant.sh`**.
- **Playwright** job (when enabled): **`pnpm/action-setup`** version **9.11.0** (aligned with root **`packageManager`**).
- Existing **`transport-quic-pipeline`** job **unchanged** in behavior (still runs on path-filtered workflow).

### E. `scripts/ci/`

| Script | Role |
|--------|------|
| **`run-full-certification.sh`** | Ordered certification: extract + H2 gate → `pnpm test` → (if not `CI=true`) Playwright strict → `make transport-lab` → `declare-readiness --strict-quic`; Docker required when not `CI=true`. |
| **`verify-docker-ports.sh`** | Assert host ports **5441–5448**, **6380** in `docker ps` output. |
| **`verify-quic-hostname-invariant.sh`** | Reject raw-IP **`BASE_URL`** defaults in **`scripts/load/**/*.js`**. |

### F. `Makefile`

- **`certify`** → **`bash scripts/ci/run-full-certification.sh`**
- **`verify-docker-ports`** → **`bash scripts/ci/verify-docker-ports.sh`**

### G. `scripts/perf/extract-protocol-matrix.js`

- **`EXTRACT_PROTOCOL_MATRIX_FROM_CSV=1`**: parse existing **`bench_logs/protocol-comparison.csv`** when no **`run-*/protocol-matrix`** directory.
- Block-comment fix (avoid `*/` inside comment that broke Node parse).

### H. `scripts/protocol/declare-readiness.js`

- Transport branch: merge **`protocol-matrix-anomalies.json`** when artifact **`protocol_integrity`** is absent or does not carry the flag (collapse gate still fires).

### I. `webapp` (strict transport + integrity)

- **`package.json`**: strict vertical / combined scripts export **`PLAYWRIGHT_STRICT_HTTP3=1`** (parity with CI when strict job runs).
- **`system-integrity.spec.ts`**: stateful checks (e.g. analytics **`new_listings`**, trust score after report, notification **`email_enabled`** toggle cycle).

## What this PR does **not** do

- Does **not** fix **HTTP/2 tail collapse** in production (that is **pool / Envoy / Caddy / backend** tuning — CI is **supposed** to stay red until **`any_http2_collapse_anomaly`** is false).
- Does **not** run **`make transport-lab`** or full **`declare-readiness --strict-quic`** on default **`ubuntu-latest`** (no cluster).
- Does **not** add **`python-ai-service`** to Docker build matrix until **`services/python-ai-service/Dockerfile`** exists in the repo.

## Reviewer FAQ (preemptive)

**Why is CI red on `protocol-anomaly`?**  
Because current committed matrix/CSV still shows **HTTP/2 collapse** (`any_http2_collapse_anomaly: true`). Merge policy: either tune H2/backends and refresh artifacts, or temporarily accept red **`och-ci`** until tuning lands (prefer fixing tuning).

**Why gate Playwright on a variable?**  
`ubuntu-latest` cannot reach private **`off-campus-housing.test`** / MetalLB. Strict HTTP/3 E2E belongs on a **self-hosted** or otherwise reachable runner.

**Why delete local-integration?**  
Architecture is **cluster-first**; the workflow suggested GitHub could substitute for k3s. CI now **does not** simulate the full app stack on Actions.

**Why require Docker for local `make certify`?**  
Downstream steps assume a **local** cluster/tooling ecosystem that uses Docker (Colima/compose/k3d patterns). **`CI=true`** skips that path.
